### Before Starting a Problem
1. Read the problem 2-3 times
2. Identify the pattern
3. Think of brute force solution first
4. Optimize step by step
5. Consider edge cases

### While Solving
1. Write pseudocode first
2. Test with examples
3. Think out loud (interview practice)
4. Time yourself (aim for 30-45 min per problem)

### After Solving
1. Review optimal solutions
2. Compare time/space complexity
3. Note alternative approaches
4. Add to pattern template

2. Set a timer for 45 minutes
3. Use Google Doc (not IDE!) to simulate whiteboard coding

   - Did you explain your approach clearly?
   - Did you consider edge cases?
